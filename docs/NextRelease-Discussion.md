# 次回リリース: 議論が必要な項目

## 目的

次回リリースに向けて方針決定が必要な項目を整理する。
`tasks.md` の「要方針」項目と連動する。

## 議論項目

### 自動更新機能と Microsoft Store 配信
- 現状: GitHub Release の更新通知を利用
- 論点: Store 版で更新通知を維持するか、Store の更新導線に寄せるか
- 決めたいこと: Store 版での表示方針とメッセージ内容
*Store版の更新導線に寄せたい MSIXの差分更新に対応*

### ファイル関連付け/ダブルクリック起動
- 対象拡張子の範囲 (jpg/png/heic など)
- MSI/Store の差分運用
- 既存のインポート/閲覧フローとの整合
決定:
- 対象拡張子は `.jpg` / `.jpeg` / `.png` / `.heic`
- 既定アプリの強制はせず、「プログラムから開く → 常に」を選べる形にする
- MSI 対応は廃止予定、Store 版のみ対応
- 既存のインポート/閲覧フローは維持 (導線は増えても可)
- ダブルクリック起動時は該当ファイルのプレビューを開き、地図も同期する
- 複数ファイル起動時はフォーカス対象のみで起動する

### Explorer 右クリックメニューの追加
- Windows 11 のコンテキストメニュー実装方式の選定
- Store 審査・署名・配布への影響
- 代替案の検討 (アプリ内導線の強化など)
*現実的な実装方法を調査。Store審査への影響が大きい場合は代替案を検討*

### 位置情報なしのピン表示ルール
- 表示しない/別枠にまとめる/通知を出す などの UX 方針
- ユーザーへの案内方法
*位置情報なし写真はフォールバック表示*
*位置情報無しを含む複数選択時にNull Islandにピンが立たないようにする*

### 収益化戦略
- 広告導入の可否 (UX/ポリシー)
- 代替案 (寄付/Pro 版など)
- 調査スコープと優先度
*ファイル表示エリアの下部にバナーを表示する案を検討*
*収益化戦略は長期的な検討課題とし、優先度は低めに設定*
*機能制限やPro版は現時点では検討しない*
*あからさまな誤タップ誘導などは避け、UXを損なわない形での広告表示を目指す*
