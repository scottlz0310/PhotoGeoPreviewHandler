## 議論項目

## 運用メモ
- Draft PR でも CI/セキュリティチェックは実行される（workflow に draft 判定がない）

### 自動更新機能と Microsoft Store 配信
- 現状: GitHub Release の更新通知を利用
- 論点: Store 版で更新通知を維持するか、Store の更新導線に寄せるか
- 決めたいこと: Store 版での表示方針とメッセージ内容
決定:
- Store 方針に合わせ、更新確認の UI は削除する

### ファイル関連付け/ダブルクリック起動
- 対象拡張子の範囲 (jpg/png/heic など)
- MSI/Store の差分運用
- 既存のインポート/閲覧フローとの整合
決定:
- 対象拡張子は `.jpg` / `.jpeg` / `.png` / `.heic`
- 既定アプリの強制はせず、「プログラムから開く → 常に」を選べる形にする
- MSI 対応は廃止予定、Store 版のみ対応
- 既存のインポート/閲覧フローは維持 (導線は増えても可)
- ダブルクリック起動時は該当ファイルのプレビューを開き、地図も同期する
- 複数ファイル起動時はフォーカス対象のみで起動する

### Explorer 右クリックメニューの追加
- Windows 11 のコンテキストメニュー実装方式の選定
- Store 審査・署名・配布への影響
- 代替案の検討 (アプリ内導線の強化など)
決定:
- 右クリックの独自メニューは追加しない
- ファイル関連付けで「プログラムから開く → 常に/一時的」を選べる導線で十分とする

### 位置情報なしのピン表示ルール
- 表示しない/別枠にまとめる/通知を出す などの UX 方針
- ユーザーへの案内方法
決定:
- 位置情報なし写真はフォールバック表示で通知する
- ピンは立てず、複数選択時に Null Island が混入して地図が広がらないようにする

### 収益化戦略
- 広告導入の可否 (UX/ポリシー)
- 代替案 (寄付/Pro 版など)
- 調査スコープと優先度
決定:
- 当面の実装計画からは外し、将来的な検討事項とする
