# CONTRIBUTING.md

## はじめに
このリポジトリに関心をお寄せいただきありがとうございます。本ファイルではコントリビューション（Issue、Pull Request、コミット、ドキュメント作成、コードコメント等）に関するルールを定めます。

## 言語ポリシー（必須）
- 本プロジェクト内のすべての対話は原則として日本語で行ってください。
  - Issue、Pull Request の説明、コメント、レビュー指摘は日本語で記載してください。
  - コミットメッセージは日本語で記述してください（例外は下記の"例外"参照）。
  - コード内のコメントおよびドキュメント（README、Wiki、API ドキュメント等）は日本語を基本とします。
- 理由：プロジェクトの参加者が一貫した言語で効率的にコミュニケーションできるようにするためです。

## 例外と英語使用ガイドライン
- 次の場合に限定して英語の使用が許可されます：
  - 外部公開APIの仕様や外部サービスのエラーメッセージ等、英語が標準の情報をそのまま記載する必要がある場合。
  - コードやライブラリが英語の識別子や仕様に依存しており、英語での説明が分かりやすい場合。
  - 国際的なコントリビューションを明示的に歓迎する場合（その際は日本語訳を併記してください）。
- 英語を使用する際は、可能な限り日本語の説明（訳）を同時に記載してください。

## Issue の書き方
- タイトルと本文は日本語で記載してください。
- 再現手順、期待される挙動、実際の挙動、環境情報（OS、.NET バージョン、利用している設定等）を明記してください。
- 再現に必要な最小限のコードやログはコードブロックで示してください。

## Pull Request (PR) の書き方
- PR の説明、変更点の要約、理由、関連する Issue 番号は日本語で記述してください。
- 影響範囲、互換性の注意点、テスト手順を明記してください。
- 大きな変更は事前に Issue で相談してください。

## コミットメッセージ
- コミットメッセージは日本語で簡潔に書いてください。
  - 例: "ログイン失敗時の例外処理を修正"
- コミットメッセージは 50 文字程度を目安に、必要なら本文で詳細を追記してください。

## コードコメントとドキュメント
- コード内部のコメント、メソッドやクラスの説明は日本語で記述してください。
- パブリックAPIのドキュメントはユーザーを意識して日本語で書きます。外部公開を前提に英語が必要な場合は英語版を別途用意し、日本語版と併記してください。

## レビューと承認
- レビュアーは日本語でフィードバックを行ってください。
- 重要な設計上の議論や決定は Issue または専用ドキュメントに日本語で記録してください。

## 自動チェック / CI
- CI で日本語固有のチェック（例: コミットメッセージの言語チェック）を導入する場合、事前にチームで合意を取ってください。

## コードスタイルとその他のルール
- 本ファイルは言語ポリシーに関するガイドラインです。コードスタイル（命名規則、インデント等）は別途 `.editorconfig` やプロジェクトのスタイルガイドに従ってください。

## 連絡先
- 本ポリシーに関する質問や提案は Issue で日本語で開いてください。
